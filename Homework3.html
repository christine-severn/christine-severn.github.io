Accessibility checklist

Website Page Titles

In the web page markup, the <title> is within the <head> tag. Good page titles are particularly important for orientation, they help people know where they are and move between pages open in their browser tab. The first thing screen readers say when the user goes to a different web page is the page title.

Results: 10 pages on the MPL website were evaluated for clear, unique HTML page titles. Each page has descriptive HTML title values. However, we do suggest that the title values have to be formatted as "Page Title at MPL", for example, "Study Rooms at MPL" so that it's obvious to users using the assistive technologies what website page they are on.

Image Text Alternatives

Image text alternatives ("alt text") (pictures, illustrations, charts, etc.) Text alternatives convey the purpose of an image, including pictures, illustrations, charts, etc. The alt attribute provides an alternative text for an image, if the user for some reason cannot view it. The reason can be a slow connection, an error with the image file, or if the user uses a screen reader.

Results: No photos on the Homepage, Events, and Spaces pages had descriptive alt text. This means that people using assistive technologies would not know what the photo or graphic is communicating. We recommend that all photos and graphics have descriptive alt text.

Text:

Headings, Contrast Ratio ("color contrast") and Resizing Text should not only support visual aesthetics but also support the navigation and visibility of text content.

Results - Headings We tested the About, Events, and Spaces pages and headings tags are not really nested in a heirarchical system. It appears headdings are used more for font functions then a heirarchical system of pages and information. The MPL homepage has 2 H1 tags, there should only be one H1 per page because screen readers expect only one H1 tag.
